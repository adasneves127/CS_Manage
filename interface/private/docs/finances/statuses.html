Finance Statuses

Financial Statuses are the method to control what stage a financial record is in the process of payment.
The statuses given by the system on creation are:
  • Pending — Used when the record has been created, but not debated or approved by either the Officers or the Student Government Financial Committee.
  • Granted — Used only for Budget records, used to indicate that the budget request has been approved by the club.
  • Denied — Used when the financial record has been denied by the Finance Committee, or denied by a vote of the officers.
  • Open — Used when an Invoice or Credit type record is awaiting payment.
  • Paid — Used when an Invoice type record has been paid, but confirmation of the items requested has not been received.
  • Closed — Used when a finance record has been paid and confirmation of the goods has been received, or all available funds from the budget have been used.
  • Cancelled — Used when an approved finance record needs to be rescinded.
This system allows Finance Administrators to modify the statuses available in the system. To do so, use the navbar on the application pages, and go to Admin -> Finance Statuses.
This page allows you to edit and create new Finance Statuses. Please note, editing a finance status will result in all records which referenced that status, updating to the new status description.
Updating the Pending status is also not recomended, as some system-automated processes are driven off of this status (Such as creation of a Docket Item on Finance Creation).
The system will also default to the Pending status for new records for non-finance admin users. If the user is a Finance Admin, then they can set the status to any existing status in the system.
